# 产品需求文档 · MediaToMD

> 一个将网页和音视频内容转换为 Markdown 的 Chrome 浏览器插件  
> **版本：** v0.2 MVP　**日期：** 2026-04　**状态：** 待评审

---

## 1. 背景与目标

### 1.1 问题背景

在互联网上看到优秀的文章、视频、音频时，保存内容非常不方便。现有工具要么格式混乱，要么依赖第三方平台存储，无法直接对接 AI 工具链。

用户希望能一键把内容转成干净的 Markdown 文件，保存到本地，方便后续检索和 AI 分析。

### 1.2 产品定位

类似 Obsidian Web Clipper，但有两个核心差异：

- 输出格式为纯 Markdown，直接保存到本地桌面
- 支持输入视频 / 音频链接，自动转录后转成 Markdown

### 1.3 目标用户

有知识管理需求、使用 AI 工具的个人用户，希望数据保存在本地、可被 AI 检索。

### 1.4 产品目标（MVP）

- 网页内容一键转 Markdown，保存到桌面
- 视频 / 音频链接自动转录，整理成结构化 Markdown，保存到桌面
- 插件内可查看历史处理记录

---

## 2. 功能范围

### 2.1 功能一：网页转 Markdown

用户在任意网页点击插件图标，当前页面正文提取为 Markdown，保存到桌面。

**规则：**
- 只提取正文内容，去除导航栏、广告、侧边栏等干扰元素
- 不保留图片，纯文字输出
- 保留原始标题层级（h1/h2/h3 → #/##/###）
- 保留链接

**输出示例：**

```markdown
# 文章标题

## 第一部分
正文内容...

## 第二部分
正文内容...
```

### 2.2 功能二：视频 / 音频转 Markdown

用户在插件内粘贴视频或音频链接，后台异步处理，完成后 Markdown 文件保存到桌面。

**处理流程：**

```
用户粘贴链接 → 提交
        ↓
yt-dlp 提取音频
        ↓
Deepgram API 转录
        ↓
Claude API 整理成结构化 Markdown
        ↓
文件保存到桌面
```

**输出格式：**

```markdown
---
title: "视频标题"
source: "https://..."
date: 2026-04-08
duration: "32:15"
---

## 摘要
（3-5 句话）

## 核心要点
- 要点一
- 要点二

## 详细内容
### 第一部分
（整理后正文）

## 原始转录
（Deepgram 原文）
```

**支持平台：** YouTube、B 站（需 cookie）及 yt-dlp 支持的其他平台

### 2.3 功能三：历史记录

插件内显示所有处理过的任务列表。

**展示信息：**
- 状态标识：✅ 成功 / ❌ 失败
- 文件标题
- 处理时间
- 失败原因（失败时显示）

**示例：**

```
✅ 2026-04-08_如何学习产品思维
✅ 2026-04-07_YC创始人访谈
❌ 2026-04-06_某视频（转录失败：链接不支持）
```

---

## 3. 文件命名规则

```
格式：YYYY-MM-DD_标题.md
示例：2026-04-08_如何学习产品思维.md
```

- 标题取网页 title 或视频标题
- 特殊字符（/ \ : * ? " < > |）替换为 -
- 标题过长时截取前 50 个字符

## 4. 技术方案

### 4.1 技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| 插件框架 | Chrome Extension Manifest V3 | 当前 Chrome 标准 |
| 网页提取 | Readability.js | Mozilla 开源，提取正文准确率高 |
| 音视频下载 | yt-dlp | 支持 1000+ 平台 |
| 语音转录 | Deepgram API | 云端转录，速度快，支持中英文 |
| AI 整理 | Anthropic Claude API | 结构化 Markdown 输出 |
| 本地保存 | Chrome Downloads API | 触发浏览器下载到桌面 |
| 历史记录 | Chrome Storage API | 本地存储，不上云 |

### 4.2 架构说明

网页转 Markdown 完全在插件本地完成，无需服务器。

视频转录需要调用外部 API，流程如下：

```
Chrome 插件
    ↓ 发送链接
后台 Service Worker
    ↓ 调用 yt-dlp（本地）提取音频
    ↓ 调用 Deepgram API 转录
    ↓ 调用 Claude API 整理
    ↓ Chrome Downloads API
保存到桌面
```

用户提交链接后可关闭插件，处理在后台进行，完成后历史记录自动更新。

### 4.3 异步处理说明

视频转录为耗时操作，采用异步方式处理：

- 用户提交后立即返回，插件不阻塞
- 历史记录中该条目显示"处理中"
- 完成后自动更新为 ✅ 或 ❌
- 用户可在处理过程中正常使用浏览器

## 5. 非功能性需求

| 指标 | 要求 |
|------|------|
| 网页转换速度 | < 3 秒完成 |
| 视频转录等待 | 用户无需等待，后台异步处理 |
| 数据隐私 | 历史记录本地存储；视频内容经过 Deepgram 和 Anthropic，仅限公开内容 |
| 文件格式 | 标准 UTF-8 编码 Markdown |
| 断网处理 | 网页转换可离线完成；视频转录需联网，断网时提示用户 |
